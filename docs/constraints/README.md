# 制約条件（Constraints）

実装過程で発見・決定された制約条件を記録するディレクトリ。

## 目的

- 設計判断とその根拠をトレーサブルにする
- 将来の開発者（人間・AI 問わず）が「なぜこうなっているか」を理解できるようにする
- 暫定措置と恒久対応の区別を明確にする

## ファイル命名規約

`{連番3桁}-{kebab-case の概要}.md`

例: `001-coordinate-precision.md`

## テンプレート

各制約条件ドキュメントには以下のセクションを含める:

```markdown
# 制約条件 NNN: タイトル

## 発生日

YYYY-MM-DD

## 対象レイヤー

影響を受けるレイヤー・パッケージ

## 制約の概要

1〜2文で制約を要約

## 背景

なぜこの制約が生じたか

## 制約詳細

具体的な制約内容（箇条書き）

## 影響範囲

影響を受けるファイル・モジュール

## 今後の対応

制約の解消・緩和に向けたアクション
```

## 記録すべきパターン

| パターン | 概要 | 例 |
|---------|------|------|
| **座標精度に関する制約** | 印刷・撮影・変換に伴う座標のずれに関する判断 | CSS mm単位のブラウザ差異、射影変換の精度限界 |
| **OCRエンジンの制約** | エンジン固有の制約事項 | NDLOCR-Liteの対応文字種、処理速度の制約 |
| **責務の線引き** | Module A/B間の責務境界 | マニフェスト生成の責務がどちらにあるか |
| **外部依存の制約** | 外部サービス・ライブラリに起因する制約 | OpenCVのバージョン制約、NDLOCR-Liteの依存関係 |
