# 未実装項目管理（Backlog）

実装予定だが未着手・着手中の項目を管理するディレクトリ。

## 目的

- 未実装項目の状態と前提条件を可視化する
- 着手可能な項目とブロックされている項目を区別する
- 担当の所有権を明確にし、並行作業の衝突を防ぐ

## 状態の定義

| 状態 | 意味 |
| --- | --- |
| 未着手 | 前提条件が未充足。着手不可 |
| 着手可能 | 前提条件が充足済み。着手可能 |
| 進行中 | 作業中。担当とブランチを明記する |
| 完了 | 実装・テスト完了 |

## ファイル命名規約

`items/{連番3桁}-{kebab-case の概要}.md`

例: `items/001-manifest-json-spec.md`

## テンプレート

各未実装項目には以下のセクションを含める:

```markdown
# Backlog NNN: タイトル

## 状態

未着手 / 着手可能 / 進行中 / 完了

## 担当

（進行中の場合）担当者名・ブランチ: `claude/xxx`

## 対象パッケージ

影響を受けるパッケージ

## 概要

何を実装するか（1〜2文）

## 未実装の理由

なぜまだ実装されていないか

## 前提条件

- [ ] 条件1
- [ ] 条件2

## 実装チェックリスト

- [ ] ステップ1
- [ ] ステップ2

## 着手時の注意事項

着手前に確認すべき事項

## 関連ドキュメント

関連する制約条件・ADR・事象レポートへのリンク
```
