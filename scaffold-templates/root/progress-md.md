# PROGRESS.md テンプレート

以下のテンプレートを `{{PROJECT_NAME}}/PROGRESS.md` として生成してください。
`{YYYY-MM-DD}` は生成日の日付で置換してください。

## テンプレートの使い方

PROGRESS.md は3つの問いに答えるファイルです:

| 問い | 答える箇所 |
|------|-----------|
| 今どうなっている？ | スナップショット部（サマリー〜次のアクション） |
| 前回から何が動いた？ | diff セクション |
| どこかで止まっていないか？ | フェーズテーブルの「最終変動」列 + 迷走検知ルール |

## フェーズの状態定義

| 状態 | 意味 |
|------|------|
| `done` | 完了 |
| `active` | 進行中 |
| `blocked` | 外部要因で止まっている |
| `waiting` | 前のフェーズの完了待ち |
| `not-started` | 未着手（着手可能な状態） |

## セクション構成ルール

- **サマリー**: 4行固定。指標名は変えない
- **フェーズ**: 5〜15行。サブフェーズは `P1-1`, `P1-2` のように枝番
- **ブロッカー**: チェックボックス形式。影響先のフェーズIDを必ず書く
- **次のアクション**: 3〜5項目（最大7）。今すぐ着手できることだけ
- **固有セクション**: プロジェクト固有の情報は「次のアクション」と「diff」の間に追加可能
- **diff**: 最新1回分を直接置き、1つ前を `<details>` で折りたたむ。最大2世代

````markdown
# PROGRESS — {{PROJECT_NAME}}

> 最終更新: {YYYY-MM-DD}
> 前回更新: {YYYY-MM-DD}

## サマリー

| 指標 | 値 |
|------|-----|
| 全体進捗 | 0% |
| 完了フェーズ | 0/{total} |
| ブロッカー | 0 件 |
| 直近の完了 | — |

## フェーズ

| ID | 名前 | 状態 | 進捗 | 最終変動 | 備考 |
|----|------|------|------|----------|------|
| P1 | （フェーズ名） | not-started | 0% | {MM-DD} | — |

## ブロッカー

（なし）

## 次のアクション

1. フェーズ構成を決定する
2. P1 の作業に着手する

## diff

> 初版作成（{YYYY-MM-DD}）

該当なし
````
